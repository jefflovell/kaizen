Neural Network Classification

Use hidden layers to bend a decision boundary, then turn final scores into class probabilities.

Business problem

Which title should a subscriber see next?

Situation

Watch intent can depend on many interacting signals: fit, recency, completion history, buzz, genre, and learned content representations.

Problem to solve

The system needs flexible class probabilities when simple thresholds or one linear boundary cannot represent those interactions.

Why a neural network

Hidden layers can learn intermediate combinations of signals before the final layer scores each possible action.

Industry contextIndustry norm for unstructured data—not every table

Neural networks dominate many image, audio, language, and large-scale representation problems because they can learn features as well as decisions. They are not automatically superior on ordinary tabular business data: random forests and boosted trees often win when datasets are smaller, features are mixed, and relationships are mostly threshold-like.

If you came through the neural-network path, this is the same machine wearing a classifier’s jacket. Features flow into hidden neurons. Each hidden neuron computes a weighted sum, adds a learned bias, applies an activation, and sends its result forward. The difference is at the end: instead of predicting a number like completion percentage, the network outputs class scores.

For a two-class problem, a sigmoid output can produce one probability. For a multi-class problem, softmax turns several scores into probabilities that add to 1. The flexibility is useful when signals interact because hidden layers give the model intermediate places to combine them. The warning is the same one from the MLP regression lab: a polished probability is only trustworthy if held-out examples show the learned boundary generalizes.

What is learned?

The weights and biases

Training adjusts the strength and offset of connections. It is searching for parameter values that reduce prediction error on labeled examples—not memorizing a hand-written rule.

Training

Learn from known answers

Repeated forward passes, loss calculations, backpropagation, and optimizer steps update the parameters. Held-out data checks whether that learning generalizes.

Inference

Use the learned parameters

After training, the weights are held fixed. A new example makes one forward pass through them to produce logits, probabilities, and a class decision.

How it is built

A neural classifier transforms, scores, and normalizes.

The training loop is still supervised learning: compare predictions to known labels, minimize loss, and update model parameters.

01 · Hidden activation
a = activation(w · x + b)

A hidden neuron combines features with weights, adds bias, then applies an activation function.

02 · Class scores
logits = W · a + b

Logits are raw output scores before sigmoid or softmax converts them to probabilities.

03 · Probability
p = softmax(logits)

Softmax converts multiple logits into class probabilities that add to 1.

Key terms

The classifier version of the network vocabulary.

Weight
A learned connection strength that determines how much one signal influences the next layer.
Logit
A raw class score before probability conversion.
Softmax
A function that converts several logits into probabilities that sum to 1.
Cross-entropy loss
A supervised loss function that penalizes confident wrong class probabilities.
Inference
Using fixed learned weights to make a prediction for a new example.
Generalization
Performance on labeled examples not used for parameter updates.
