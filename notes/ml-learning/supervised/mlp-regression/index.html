Applying a Neural Network

Use a multilayer perceptron to test when nonlinear feature combinations earn trust, and when they only produce a confident-looking number.

Congratulations! At this point, you have already done the hard work. In Building a Neural Network, you learned how raw facts become a feature vector, how weights and bias shape a neuron, and why hidden layers can move deep neural networks past simple one-step prediction into something that feels more human. In Training a Neural Network, you learned how the machine learns: using loss functions to update those parameters automatically, instead of requiring explicit hard-coding and constant human adjustment. In this lesson, we will finally stop bolting overnight parts from Japan onto the 2JZ and start living one quarter-mile at a time.

This page uses the same hardware that you've learned to ask an applied question: when does a neural network make a better numeric prediction than a straight-line model, and when is that prediction less trustworthy than it looks?

For example, in the real world we might ask, "How much of a newly released streaming title will this group of subscribers finish during its first week if we make sure to show it to them as soon as they open the app?" Audience–title affinity matters—but its effect may change significantly with title awareness, release timing, runtime, tone preference, and social momentum.

Unlike the simple perceptron we used for the first classification lesson, a multilayer perceptron (MLP) passes those features through additional, hidden layers of neurons. Each neuron in these hidden layers computes a weighted sum, adds a learned bias parameter, applies an activation strength, and sends its result forward. Together, those hidden activations can bend the signal to represent complex relationships between different features that a single linear equation misses, creating a prediction that feels richer and more nuanced, even on new data.

Applied reason

Why use a DNN at all?

A straight-line model is a good place to begin because it makes a clean promise: each feature gets one steady push on the prediction. More affinity pushes the number up by this much. More runtime pulls it down by that much. That simplicity is useful, but it can also become the whole problem.

Real audience behavior does not always move one feature at a time. Affinity may matter more after awareness is already high. A long runtime may hurt a casual comedy and barely matter for a prestige drama. Social momentum may amplify tone preference instead of adding a separate, tidy amount. Once signals start changing the meaning of other signals, a single straight line has no place to hold that intermediate relationship.

A deep neural network adds those intermediate places. Hidden neurons combine features, apply activation functions, and pass learned signals forward before the final prediction is made. Training adjusts the weights and bias terms so those combinations fit the examples used for updates, then held-out examples test whether the pattern still works on data the model did not train on.

That does not make a DNN magic, or automatically better than random forests, linear regression, or anything else in the toolbox. It just gives the model a different kind of flexibility: smoother than split rules, less rigid than one linear equation, and powerful enough to bend when the data really asks for a bend. The work of this page is to see when that extra flexibility is trustworthy and when it is a liability.

Linear vs nonlinear

Choosing the right activation shape.

Quick reminder from the Build lesson: z is the neuron's pre-activation value, the weighted sum before the activation function runs. a is the activation, the value that leaves the neuron and becomes input for the next layer. The activation function is the rule that turns z into a.

If the activation is linear, the neuron passes its weighted sum through unchanged. That is useful as a control because it keeps the same network wiring while asking a simpler question: what can the features explain if every signal is allowed to push the prediction by a steady amount?

A nonlinear activation changes that rule. ReLU, for example, creates a hard bend at zero: weak or negative pre-activation values send out nothing, while positive values pass forward. That bend gives a hidden neuron a way to act like a learned threshold. Audience affinity might not matter much until awareness is high enough; after that point, the same affinity signal can begin to matter a lot.

The other shapes are here to make the choice visible. Leaky ReLU keeps a small signal alive below zero. Sigmoid and tanh compress strong values into flatter ends, which can be useful when a signal should saturate instead of growing forever. In this lab, we mainly compare the blue straight line with the purple ReLU bend because that contrast shows the core tradeoff: steady addition versus a learned place where the model can turn.

How it is built

Combine. Activate. Update.

The lab uses one hidden layer, squared error for a single selected profile, and Mean Squared Error (MSE) for training and held-out sets. Every visible prediction comes from these operations.

01 · Combine the inputs
z = Σ(wixi) + b

Σ (“capital sigma”) means add the terms. wi (“w sub i”) is a weight, xi is input i, and b is the bias.

02 · Apply an activation
a = ReLU(z) = max(0, z)

a is the neuron’s activation. ReLU (Rectified Linear Unit) keeps positive values and turns negative values into zero.

03 · Update a parameter
w ← w − η(∂L/∂w)

η (“eta”) is the learning rate. ∂L/∂w (“partial L over partial w”) measures how changing this weight changes the loss.

Compact layer notationa = f(Wx + b)

This performs the same weighted-sum-and-activation calculation for every hidden neuron at once. Matrix notation compresses the scalar process—it does not replace it.

Key terms

The vocabulary of learned representations.

A network is built from simple operations; the depth comes from composing and training them together.

Hidden layer
Neurons between the raw inputs and final output that learn intermediate features.
Weight
A learned multiplier controlling the strength and direction of a connection.
Bias
A learned offset that lets a neuron shift when it becomes active.
Forward pass
Calculating activations from the inputs through to the prediction.
Backpropagation
The efficient backward pass that calculates gradients for the parameters.
Gradient
The direction and sensitivity of change in the loss.
Epoch
One training pass through every example in the training set.
Hyperparameter
A setting chosen outside training, such as learning rate or hidden-neuron count.
