Regularized Linear Models

Ridge and Lasso deliberately trade a little training fit for a simpler model that can perform better on new examples.

Imagine predicting first-week completion for a new streaming title from hundreds of signals: tone, cast, franchise history, runtime, release timing, social buzz, and learned embeddings. Ordinary linear regression may assign confident weights to patterns that happened by accident in the training catalog.

Each signal gets a coefficient: a learned weight describing how strongly—and in which direction—that feature changes the prediction. Regularization adds a cost for large coefficients. Ridge gently shrinks every weight; Lasso can remove weak features entirely. Both ask the model to explain the data with less complexity.

  • Ridge keeps every feature but reduces its influence.
  • Lasso can set weak coefficients exactly to zero.

How it is built

The model fits the data and pays for complexity.

A loss function is the score the training process tries to minimize: lower means the current model is doing a better job under the rules we gave it. Ridge and Lasso begin with Mean Squared Error, then add a penalty controlled by λ (“lambda”).

01 · Measure prediction error
MSE = 1n Σi=1n(yi − ŷi)2

A residual is the gap between an actual result yi and its prediction ŷi (“y-hat sub i”). MSE (Mean Squared Error) squares those gaps, then averages them across all n examples.

02 · Ridge adds an L2 penalty
LossRidge = MSE + λΣj=1mbj2

Ridge squares each coefficient bj (“b sub j”). Large weights become expensive, so every coefficient is smoothly pulled toward zero.

03 · Lasso adds an L1 penalty
LossLasso = MSE + λΣj=1m|bj|

Lasso uses absolute values. Its geometry can make the cheapest solution land exactly at zero, effectively removing weak features.

Key terms

The vocabulary of controlled complexity.

Regularization is less about making the training score perfect and more about building a rule that survives new data.

Regularization
Adding a complexity penalty to the model's training objective.
Coefficient
A weight learned for a feature. Its size controls the feature's influence; its sign controls whether the prediction moves up or down.
Residual
The difference between an observed value and the model's prediction: actual minus predicted.
Loss function
A single score describing how costly the model's mistakes are. Training searches for parameter values that make this score smaller.
λ (lambda)
The hyperparameter that sets how strongly the loss function penalizes large coefficients.
Ridge / L2
A squared-coefficient penalty that shrinks every feature smoothly.
Lasso / L1
An absolute-value penalty that can force weak coefficients to zero.
Coefficient shrinkage
Reducing learned weights so the model relies less on fragile patterns.
Generalization
How well a model performs on examples it did not see during training.
Hyperparameter
A learning-rule setting chosen by the practitioner—not learned as a coefficient from the training examples. We compare candidate settings, such as several values of λ, using held-out data.
Feature selection
Keeping useful inputs while excluding weak ones; Lasso can do this automatically.
