Logistic Regression as a Classifier

Use a straight-line score when you need a category decision and a probability you can inspect.

Business problem

Which messages need review—and how sure are we?

Situation

A moderation queue receives far more messages than people can inspect manually.

Problem to solve

The team needs both a review decision and a score it can threshold differently as risk and staffing change.

Why logistic regression

It turns a fast linear score into an inspectable probability, making the action threshold explicit.

Industry context Industry-standard baseline

Logistic regression remains a normal production choice for tabular classification when latency, explainability, probability calibration, or regulatory review matter. More complex models should prove they beat this baseline by enough to justify their added cost.

The perceptron gave us a first classifier: score the example, guess a side, and update the boundary after mistakes. Logistic regression keeps the straight-line score, but it adds an important layer of interpretation. Instead of only saying “this side or that side,” it squeezes the score into a probability between 0 and 1.

The name is annoying on purpose, or at least it feels that way. Logistic regression is usually used for classification, not numeric regression. The “regression” part comes from the model’s linear score: it still computes a weighted sum like a regression model. The “logistic” part is the S-shaped curve that converts that score into a probability. So the name really means: a regression-style score passed through a logistic curve so it can support a class decision.

That matters when the decision has a cost. If a moderation model is 51% sure a message needs review, you may route it differently than a message at 97%. The boundary is still linear, but the readout now says how far the example sits from that boundary in probability language. The useful question becomes: is this probability calibrated, or is it just a clean-looking number?

What to watch

  • Raising weighted evidence moves z to the right on the sigmoid curve.
  • The sigmoid converts that score into probability, but the curve flattens near 0 and 1.
  • The threshold controls the action. It does not retrain the model.
  • Trust comes from calibration on held-out or recent examples, not from a clean probability display.

Stats bridge

From least-squares lines to trained classifiers.

If you have taken college statistics, you may remember least-squares linear regression: measure the residuals, square them, add them up, and choose the line that makes that total as small as possible. In the simplest ordinary least-squares setting, the algebra can solve directly for the best line. That is the clean old friend hiding behind y = mx + b.

Logistic regression expands that idea so the line can support classification. It still builds a linear score, z = w · x + b, but it does not use that score as the final numeric prediction. It passes the score through the logistic sigmoid so the readout lands between 0 and 1, then trains the weights and bias against classification loss instead of squared-error loss.

That extra sigmoid-and-loss step is why this lesson starts to feel like the bridge into neural networks. We are no longer just drawing the line with one tidy least-squares formula. Modern implementations use linear algebra to compute many scores and gradients at once, then an optimizer changes the parameters until the loss gets smaller. Deep neural networks scale up the same habit: weighted sums, nonlinear functions, loss, gradients, and parameter updates.

How it is built

Logistic regression scores, squashes, and thresholds.

The model is linear until the final probability conversion. Training usually minimizes log loss, which strongly penalizes confident wrong probabilities.

01 · Linear score
z = w · x + b

x is the feature vector, w is the weight vector, and b is the learned bias offset.

02 · Probability
p = 1 / (1 + e-z)

The sigmoid function turns any score into a value between 0 and 1. Large positive scores move toward 1; large negative scores move toward 0.

03 · Class decision
ŷ = 1 if p ≥ t

ŷ (“y-hat”) is the predicted class. t is the threshold, often 0.5 unless the cost of mistakes suggests otherwise.

Key terms

Words that make the probability honest.

Logit
The raw linear score before the sigmoid function.
Sigmoid
An S-shaped function that maps a score to a probability-like value.
Threshold
The cutoff that turns probability into a class action.
Calibration
Whether examples predicted near 80% really happen about 80% of the time.
