Decision Tree Regression

A regression tree asks a sequence of threshold questions, then predicts one numeric value for each resulting region.

Linear regression describes the whole market with one line. A decision tree takes another approach: split the examples into groups, then predict the average outcome inside each group.

This makes trees good at capturing jumps, plateaus, and other nonlinear patterns. It also creates a danger: a deep tree can keep splitting until it learns the quirks of individual listings instead of the broader market.

Splits

A split asks a yes-or-no question such as performance < 121?. The tree chooses the threshold that reduces squared prediction error the most.

Leaves

A leaf is an endpoint in the tree. Every GPU that reaches the same leaf receives the same predicted price: the average sale price of the training examples in that region.

Depth

Depth limits how many questions the tree may ask on the way to a prediction. Shallow trees are simpler; deeper trees can represent finer price bands.

Overfitting

If a tree grows too deep, it can isolate individual sales and achieve tiny training error while making brittle predictions for new listings.
