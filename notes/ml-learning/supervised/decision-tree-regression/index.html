Decision Tree Regression

A regression tree asks a sequence of threshold questions, then predicts one numeric value for each resulting region.

Linear regression describes the whole market with one line. A decision tree takes a deeper approach: split the examples into groups, then predict the average outcome inside each group.

This makes trees good at capturing jumps, plateaus, and other nonlinear patterns. It also creates a danger: a deep tree can keep splitting until it learns the quirks of individual listings instead of the broader market.

How it is built

A tree repeatedly chooses the least-wrong split.

At each node, the algorithm tests candidate questions, measures the error on both resulting groups, and keeps the split that reduces squared error the most.

01 · Score a candidate split
E(s) = Σi ∈ L(s)(yi − ȳL)2 + Σi ∈ R(s)(yi − ȳR)2

A candidate question s divides examples into left and right groups. Σ (“capital sigma”) means “sum,” and each group is scored against its average target ȳ (“y-bar”).

02 · Keep the best question
s* = arg mins E(s)

Test the available feature thresholds and select the split s* (“s-star”) with the lowest combined error. arg min means “the input that produces the minimum.” Then repeat inside each new branch.

03 · Predict from a leaf
ŷR = 1|R| Σi ∈ R yi

When splitting stops, every example in leaf R receives prediction ŷR (“y-hat sub R”). |R| (“the size of R”) is its example count, and Σ adds their targets before averaging.

Key terms

The vocabulary of branching decisions.

Trees replace one global equation with a hierarchy of local questions and regional predictions.

Node
A location in the tree that either asks a question or returns a prediction.
Split
A yes-or-no threshold question that divides examples into two branches.
Threshold
The cutoff value in a question such as performance < 121?.
Branch
One path created by the answer to a split question.
Leaf
An endpoint whose prediction is the average target of the examples inside it.
Depth
The maximum number of split questions one prediction encounters from root to leaf.
Greedy search
Choosing the best available split now without reconsidering the whole tree later.
Overfitting
Memorizing training quirks so precisely that predictions become worse on new examples.
