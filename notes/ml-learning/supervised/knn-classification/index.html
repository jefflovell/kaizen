k-Nearest Neighbors Classification

Classify a new example by asking the labeled examples closest to it to vote.

k-nearest neighbors, usually shortened to k-NN, has almost no learned internal machinery. It stores labeled examples. When a new example arrives, the model measures distance, finds the k closest neighbors, and predicts the class that wins the local vote.

That makes k-NN concrete in a way beginners can see: the model is trusting local precedent. It is useful when similar examples really should receive similar labels, especially in pockets that a single straight boundary would miss. It becomes fragile when “nearby” is only a math accident caused by badly scaled or poorly chosen features.

01

Store examples

Keep labeled training examples in feature space.

02

Measure distance

Find which examples are closest to the new point.

03

Vote locally

Predict the class most common among the nearest neighbors.

Draft lab | Ticket routing

Nearby tickets vote on the route.

Imagine routing support tickets into billing, technical, or account queues. Instead of fitting one global equation, k-NN asks: which past tickets look most like this one? The answer depends on the feature scale, the distance formula, and the value of k. A good k-NN setup makes meaningful similarity visible; a bad one lets irrelevant numeric size decide the vote.

What to watch

  • There is no fitted line or tree to inspect.
  • k controls how local the vote is.
  • Feature scaling changes who counts as nearby.
  • Dense neighborhoods are usually safer than lonely points.

How it is built

k-NN turns similarity into a vote.

The model’s “training” is mostly storing examples. The important design choices happen in the distance measure, feature scaling, and neighbor count.

01 · Distance
d(x, xᵢ)

Measure how far the new feature vector x is from training example xᵢ (“x sub i”).

02 · Neighbor set
Nₖ(x)

Nₖ is the set of the k closest labeled examples.

03 · Vote
ŷ = mode(yᵢ)

The predicted class ŷ is the most common label among the neighbors.

Key terms

The vocabulary of nearby.

k
The number of neighbors allowed to vote.
Distance metric
The formula that decides what “near” means.
Feature scaling
Putting features on comparable numeric ranges before distance is measured.
Local decision
A prediction based on nearby examples, not one global rule.
