Naive Bayes Classification

Compare which class best explains the evidence by multiplying simple likelihoods.

Naive Bayes is fast because it makes a bold simplification. It estimates how often each feature appears in each class, then combines those feature likelihoods as if the features were independent once the class is known. That independence assumption is the “naive” part.

The simplification is often wrong in the literal world and still useful in the modeling world. Words in an email are not truly independent, but a spam filter can still learn that certain words, senders, and link patterns are much easier to explain under “spam” than under “not spam.” The model earns trust when those likelihood estimates come from enough representative examples.

What to watch

  • Priors matter most when the feature evidence is weak.
  • Likelihoods are estimated separately for spam and not-spam.
  • The independence assumption can overcount correlated clues.
  • Smoothing prevents unseen features from erasing a class score.

How it is built

Naive Bayes compares explanations.

The classifier scores each possible class by combining the class prior with the likelihood of the observed features under that class.

01 · Prior
P(y)

The prior is how common class y is before looking at the current features.

02 · Likelihoods
P(xᵢ | y)

This asks how often feature xᵢ appears when the class is y.

03 · Compare scores
ŷ = argmax P(y) Π P(xᵢ | y)

argmax means choose the class with the largest score. Π (“capital pi”) means multiply the feature likelihoods.

Key terms

The vocabulary of evidence.

Prior
The starting probability of a class before reading the features.
Likelihood
How probable a feature is if a class were true.
Independence assumption
The simplifying claim that features can be combined separately within a class.
Smoothing
A small adjustment that prevents unseen features from making a class score zero.
