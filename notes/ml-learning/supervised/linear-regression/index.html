Linear Regression

Linear regression learns a straight-line relationship between a numeric input and a numeric outcome.

Regression is the branch of supervised learning that predicts a number rather than a category. Here, the input is a GPU's gaming performance and the outcome is its used-market price.

A fitted line gives us a useful baseline: what would a GPU normally cost at this performance level? Real listings wander above and below it because price also reflects age, VRAM, condition, efficiency, scarcity, and the occasional wildly optimistic seller.

From benchmarks to a feature

Many frame rates become one comparable number.

Raw evidence might start as average frames per second across several games tested at the same resolution and graphics settings. Because one game may favor a particular GPU, a benchmark suite combines results from many games instead of trusting a single frame-rate result.

We normalize those results against a reference GPU. In this simplified example, the reference card receives a relative performance score of 100. A card that is roughly 50% faster across the benchmark suite would score near 150.

That single score becomes the model's input feature, while used price is the target it tries to predict. This index is a teaching abstraction, not an industry standard. A real pricing model might retain game-specific results alongside VRAM, age, condition, efficiency, and warranty instead of compressing performance into one number.

One training epoch

  1. Predict the price of every GPU.
  2. Measure each residual: actual minus predicted.
  3. Adjust the line to reduce the combined error.
  4. Repeat until the line settles near the best fit.

What to watch

  • Dots are known sales used for training.
  • The line predicts price from performance.
  • Residuals are the gaps between dots and line.
  • Outliers can tilt the model away from the trend.

Slope

The slope describes how much price tends to rise when performance increases. A steeper line means the market pays a larger premium for each additional unit of gaming performance.

Intercept

The intercept anchors the line. It is the model's predicted price when the normalized performance input is zero—not necessarily a useful real-world GPU, but a number the equation needs.

Residuals

A residual is the difference between an actual sale and the model's estimate. A card above the line sold for more than expected; one below it sold for less.

Missing features

Performance alone cannot explain every price. Adding age, VRAM, condition, power use, and warranty would turn this into multiple linear regression and usually improve the estimate.
