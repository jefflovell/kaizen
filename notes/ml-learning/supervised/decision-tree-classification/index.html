Decision Tree Classification

Ask a sequence of feature questions until an example lands in a labeled leaf.

Business problem

Which accounts need human risk review?

Situation

An account can look safe under one condition and risky under another, so one global score misses important interactions.

Problem to solve

Reviewers need a consistent decision and a route they can explain: which questions sent this account to this outcome?

Why a decision tree

A tree learns threshold questions that split the feature space into understandable regions with different class mixes.

Industry contextInterpretable rules; ensembles for accuracy

Single trees are useful when a transparent rule path matters, but they are unstable enough that production teams often prefer random forests or boosted trees. The single tree is the foundation; the ensemble is usually the stronger tabular predictor.

A decision tree feels close to a checklist: if this feature is high, go right; otherwise go left. Keep asking questions until the example reaches a leaf, then predict the class most common in that leaf.

The useful part is also the danger. Trees can discover sharp interactions without asking for a straight line: a login velocity that looks safe for one account type may be risky for another. But a deep tree can also memorize training quirks. The practical question is not “can the tree split?” It is “which splits keep working on examples that did not shape the tree?”

What to watch

  • Each node asks one feature question.
  • Changing a slider can send the account down a different path.
  • Leaves contain a class distribution, not magic certainty.
  • Reducing max depth prunes the path and can make the answer less memorized.

How it is built

A tree routes examples into labeled regions.

Training searches for splits that make child nodes more class-pure than their parent. Prediction follows the learned questions to a leaf.

01 · Split rule
xⱼ ≤ c?

Ask whether feature xⱼ (“x sub j”) is below cutoff c.

02 · Reduce impurity
ΔI = I(parent) - I(children)

The chosen split should reduce impurity, meaning the child nodes have cleaner label mixes.

03 · Leaf vote
ŷ = majority(leaf)

The predicted class is the most common training label in the reached leaf.

Key terms

The vocabulary of splits.

Node
A decision point that asks one feature question.
Leaf
A terminal region that stores the class mix used for prediction.
Impurity
A measure of how mixed the labels are inside a node.
Pruning
Limiting or simplifying the tree to reduce overfitting.
