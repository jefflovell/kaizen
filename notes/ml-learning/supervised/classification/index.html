Classification

Classification is the supervised learning task of turning measured facts into a category decision.

Classification is the other half of the supervised-learning doorway. Regression asks for a number. Classification asks for a decision: spam or not spam, fraud or legitimate, churn risk or healthy account, cyan or coral. The model still reads features, compares predictions with known labels, and updates from mistakes, but the output is a category instead of a numeric estimate.

This first classifier is deliberately simple: a perceptron learns one straight decision boundary. That line is not the end of classification; it is the first object worth understanding. Once the line makes sense, later classifiers can change the mechanism: estimate probability, vote from neighbors, split through a tree, widen a margin, or combine many weak decisions.

01

Train on labels

Start with examples that already include the right answer.

02

Adjust after mistakes

Compare each guess with the label and nudge the rule.

Predict without labels

Use the learned rule on new examples where the answer is unknown.

Classification algorithms: Every classifier answers the same category question, but each one earns trust through a different mechanism.

01

Perceptron

Score a point, guess a side, and move a straight boundary only when the guess is wrong.

02

Logistic Regression

Turn a linear score into a probability, then choose a class with a threshold.

03

k-Nearest Neighbors

Find nearby labeled examples and let the local neighborhood vote.

04

Naive Bayes

Combine feature likelihoods to compare which category explains the evidence best.

05

Decision Trees

Ask a sequence of feature questions until an example lands in a labeled leaf.

06

Support Vector Classifier

Find a separating boundary and make the margin around it as wide as possible.

07

Ensembles

Combine many imperfect decisions so the final vote is steadier than one model alone.

08

Neural Networks

Pass features through hidden layers so the boundary can bend when signals interact.

This interactive graph demonstrates a perceptron, one of the earliest linear classification algorithms. “Linear” means it tries to separate the two groups with one straight line. The line is the model's current decision boundary. Points on one side are guessed as cyan; points on the other side are guessed as coral. Each point has a known label. When the model guesses wrong, the update changes the weights and bias so that the boundary moves toward putting that example on the correct side.

How it is built

A perceptron scores, guesses, and corrects.

Each labeled example passes through the same three calculations. Correct guesses leave the boundary alone; mistakes adjust its weights and bias.

01 · Calculate a score
z = w · x + b

Multiply the feature vector x (“x”) by the weights w (“w”), then add the bias b (“b”). The sign and size of score z (“z”) describe which side of the boundary the point occupies. Dividing by ||w|| (“the magnitude of w”) converts it to the signed distance shown in the live trace.

02 · Convert score to a class
ŷ = sign(z)

A nonnegative score predicts cyan (ŷ = 1); a negative score predicts coral (ŷ = −1). The symbol ŷ is pronounced “y-hat” and means the prediction. Compare it with known label y (“y”).

03 · Correct a mistake
w ← w + ηyx
b ← b + ηy

Only a wrong guess triggers an update. The learning rate η (“eta”) controls the step size; the label y controls the direction of the nudge.

Key terms

The vocabulary behind the motion.

These terms recur throughout machine learning. The graph makes them visible; the definitions make them portable.

Feature vector
An ordered list of input values. Here, x = [horizontal position, vertical position].
Weights
Tuneable coefficients that determine how strongly each feature affects the score.
Bias
An adjustable offset that lets the boundary move without changing its angle.
Learning rate
η (“eta”) controls how far the boundary moves after each mistake.
Label and prediction
y is the known answer used in training; ŷ (“y-hat”) is the model's guess.
Decision boundary
The line where w · x + b = 0; its two sides produce different classes.
Epoch
One complete pass through every training example: guess, compare, and adjust.
Outlier
An unusual example far from the broader pattern that can pull the learned boundary off course.
