Classification

Classification is the supervised learning task of turning measured facts into a category decision.

Business problem

Can one consistent rule sort incoming cases?

Situation

An operations team has labeled historical cases and measured features for each one.

Problem to solve

New cases need a fast binary decision without hand-writing every possible rule.

Why a perceptron

It makes learning visible: each mistake nudges one straight decision boundary toward a better separation.

Industry context Foundational and learning-first

The perceptron is historically important and excellent for understanding learned weights. Modern production teams more often choose logistic regression or a linear support-vector model because they provide probabilities, margins, and mature evaluation workflows.

Classification is the other half of the supervised-learning doorway. Regression asks for a number. Classification asks for a decision: spam or not spam, fraud or legitimate, churn risk or healthy account, cyan or coral. The model still reads features, compares predictions with known labels, and updates from mistakes, but the output is a category instead of a numeric estimate.

If this is your first classifier, start with k-nearest neighbors: place a new example near labeled examples and let the local neighborhood vote. It makes the input, evidence, and decision visible without requiring a training rule first. Then return to the perceptron lab below to see how a model can learn one global linear boundary from mistakes.

Optional deep dive | Linear classification

A perceptron learns a boundary.

One training step

  1. Read one labeled point.
  2. Guess which side of the line it belongs on.
  3. Compare the guess with the known label.
  4. If wrong, nudge the line toward the right answer.
if wrong: w ← w + ηyx; b ← b + ηy

This page uses the common y ∈ {−1, +1} label convention. Perceptrons written with 0/1 labels use a different-looking update, but the same idea: only mistakes move the boundary. An epoch is one full pass through every point.

This interactive graph demonstrates a perceptron, one of the earliest linear classification algorithms. “Linear” means it tries to separate the two groups with one straight line. The line is the model's current decision boundary: crossing it changes the predicted class. At reset, everything to its left is predicted coral and everything to its right is predicted cyan. Training can rotate or shift the line, but the rule remains the same: the tinted half-plane on each side maps to the class named in the legend. Each dot's color is its known label, which may disagree with the background prediction.

What to watch

  • Dot color is the known training label.
  • The white line is the decision boundary: crossing it flips the guess.
  • The coral and cyan washes show the class predicted on each side.
  • Score shows signed distance from the line.
  • A white ring marks a mislabeled side; mistakes move the line.

How it is built

A perceptron scores, guesses, and corrects.

Each labeled example passes through the same three calculations. Correct guesses leave the boundary alone; mistakes adjust its weights and bias.

01 · Calculate a score
z = w · x + b

Multiply the feature vector x (“x”) by the weights w (“w”), then add the bias b (“b”). The sign and size of score z (“z”) describe which side of the boundary the point occupies. Dividing by ||w|| (“the magnitude of w”) converts it to the signed distance shown in the live trace.

02 · Convert score to a class
ŷ = sign(z)

A nonnegative score predicts cyan (ŷ = 1); a negative score predicts coral (ŷ = −1). The symbol ŷ is pronounced “y-hat” and means the prediction. Compare it with known label y (“y”).

03 · Correct a mistake
w ← w + ηyx
b ← b + ηy

Only a wrong guess triggers an update. The learning rate η (“eta”) controls the step size; the label y controls the direction of the nudge.

Key terms

The vocabulary behind the motion.

These terms recur throughout machine learning. The graph makes them visible; the definitions make them portable.

Feature vector
An ordered list of input values. Here, x = [horizontal position, vertical position].
Weights
Tuneable coefficients that determine how strongly each feature affects the score.
Bias
An adjustable offset that lets the boundary move without changing its angle.
Learning rate
η (“eta”) controls how far the boundary moves after each mistake.
Label and prediction
y is the known answer used in training; ŷ (“y-hat”) is the model's guess.
Decision boundary
The line where w · x + b = 0; its two sides produce different classes.
Epoch
One complete pass through every training example: guess, compare, and adjust.
Outlier
An unusual example far from the broader pattern that can pull the learned boundary off course.
