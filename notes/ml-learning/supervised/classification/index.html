Classification

Classification is the supervised learning task of assigning new examples to categories learned from labeled examples.

Classification is one of the two foundational supervised learning tasks. It teaches a model which details about an example matter when deciding which category it belongs to. The trained model that makes those decisions is called a classifier.

Classifiers are useful whenever the answer is a category: deciding whether an email is spam, identifying an object in a photo, detecting fraud, or guessing whether a new example belongs to group A or group B. Other supervised learning models solve different tasks, such as regression models that predict numbers.

Classification algorithms: Different model families learn categories, neighborhoods, probabilities, or decision boundaries in different ways.

One training step

  1. Read one labeled point.
  2. Guess which side of the line it belongs on.
  3. Compare the guess with the known label.
  4. If wrong, nudge the line toward the right answer.

This interactive graph demonstrates a perceptron, one of the earliest linear classification algorithms. “Linear” means it tries to separate the two groups cleanly with a single straight line. The line is the machine's current decision boundary: points on one side are guessed as cyan, and points on the other side are guessed as coral. Each point has an attached label. If the machine guesses the wrong color because of the line's position relative to the point, the position and slope of the line are updated to attempt to put that point on the correct side of the line.

What to watch

  • Dots are labeled examples.
  • The white line is the decision boundary.
  • Score shows signed distance from the line.
  • Mistakes nudge the line toward better guesses.

Features

Each training example in this demonstration (a dot on the graph) is simply a list of numbers and a label: a horizontal position (x-axis), a vertical position (y-axis), and a label with a number that means “cyan” or “coral,” like 1 or -1. These values are called features, and the list of these feature values is called a feature vector. All feature vectors, or training examples, have the same number of features in the same order.

Weights

The equation w ← w + η(y - ŷ)x says: if the guess is wrong, adjust the machine's weights to redraw the boundary line. You can think of weights like tuneable knobs that the machine can use to boost or cut how important a feature or a specific feature value is. w is the current set of weights, η (eta) is the learning rate, or how big of an adjustment to make after each wrong guess, y is the label of the example (cyan or coral), ŷ (y-hat) is the machine's guess, and x is the training example.

Epochs

The machine looks at each training example (feature vector) and makes a guess, then adjusts the line based on wrong answers, trying to improve the accuracy of guesses as much as possible. The machine repeats this process over and over without understanding anything about the data. It simply tries to make the best fit through iterative training: guess, compare, adjust, and repeat. Training uses labels. Prediction uses the learned rule on new examples without labels. One full pass through all of the training examples is called a training epoch. For real-world problems, which frequently have hundreds or thousands of features and millions of training examples, it can take thousands or more training epochs to get a useful model. That's billions or trillions or even more guesses.

Outliers

Step through the model or train a few epochs until the model accuracy hits 100%. Then try placing a cyan dot or three in the cluster of coral dots, or stick some dots way off on the edges of the graph and see how the model responds to more training. These unusual kinds of examples that do not fit the rest of the data are called outliers, and they are one of the main challenges of real-world problems where data is messy.
