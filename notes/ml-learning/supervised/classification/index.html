Classification

Classification is the supervised learning task of assigning new examples to categories learned from labeled examples.

Classification is one of the two foundational supervised learning tasks. It teaches a model which details about an example matter when deciding which category it belongs to. The trained model that makes those decisions is called a classifier.

Classifiers are useful whenever the answer is a category: deciding whether an email is spam, identifying an object in a photo, detecting fraud, or guessing whether a new example belongs to group A or group B. Other supervised learning models solve different tasks, such as regression models that predict numbers.

01

Train on labels

Start with examples that already include the right answer.

02

Adjust after mistakes

Compare each guess with the label and nudge the rule.

03

Predict without labels

Use the learned rule on new examples where the answer is unknown.

This interactive graph demonstrates a perceptron, one of the earliest linear classification algorithms. “Linear” means it tries to separate the two groups cleanly with a single straight line. The line is the machine's current decision boundary: points on one side are guessed as cyan, and points on the other side are guessed as coral. Each point has an attached label. If the machine guesses the wrong color because of the line's position relative to the point, the position and slope of the line are updated to attempt to put that point on the correct side of the line.

How it is built

A perceptron scores, guesses, and corrects.

Each labeled example passes through the same three calculations. Correct guesses leave the boundary alone; mistakes adjust its weights and bias.

01 · Calculate a score
z = w · x + b

Multiply the feature vector x (“x”) by the weights w (“w”), then add the bias b (“b”). The sign and size of score z (“z”) describe which side of the boundary the point occupies. Dividing by ||w|| (“the magnitude of w”) converts it to the signed distance shown in the live trace.

02 · Convert score to a class
ŷ = sign(z)

A nonnegative score predicts cyan (ŷ = 1); a negative score predicts coral (ŷ = −1). The symbol ŷ is pronounced “y-hat” and means the prediction. Compare it with known label y (“y”).

03 · Correct a mistake
w ← w + ηyx
b ← b + ηy

Only a wrong guess triggers an update. The learning rate η (“eta”) controls the step size; the label y controls the direction of the nudge.

Key terms

The vocabulary behind the motion.

These terms recur throughout machine learning. The graph makes them visible; the definitions make them portable.

Feature vector
An ordered list of input values. Here, x = [horizontal position, vertical position].
Weights
Tuneable coefficients that determine how strongly each feature affects the score.
Bias
An adjustable offset that lets the boundary move without changing its angle.
Learning rate
η (“eta”) controls how far the boundary moves after each mistake.
Label and prediction
y is the known answer used in training; ŷ (“y-hat”) is the model's guess.
Decision boundary
The line where w · x + b = 0; its two sides produce different classes.
Epoch
One complete pass through every training example: guess, compare, and adjust.
Outlier
An unusual example far from the broader pattern that can pull the learned boundary off course.
