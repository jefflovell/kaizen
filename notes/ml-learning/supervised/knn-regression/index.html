k-Nearest Neighbors Regression

k-NN predicts a number by finding the most similar known examples and averaging their outcomes.

Imagine a new streaming title entering the catalog with no viewing history. A global model could apply one rule to every title, and a tree could place it into a predefined segment. k-NN takes a more local approach: find titles that resemble this one and borrow their signal.

This is useful during cold start, when behavioral data does not exist yet. Content features—tone, pace, genre, runtime, audience, cast, or learned embeddings—provide an initial basis for similarity until real viewing behavior arrives.

    What to watch

    • Small k reacts strongly to individual titles.
    • Large k produces smoother, more generic estimates.
    • The neighborhood changes for every query.
    • Distance only works when features share a sensible scale.

    Cold start

    New titles have no clicks, plays, completions, or watch time. Content-based neighbors provide a temporary prediction until enough behavioral evidence exists to personalize more directly.

    Choosing k

    With k = 1, one unusual neighbor can dominate. As k grows, the estimate becomes more stable but less specific. The best value is a model choice validated against held-out examples.

    Feature scaling

    Distance treats every numeric dimension literally. A runtime measured from 20–200 could overpower a tone score measured from 0–1, so production features must be normalized or otherwise weighted deliberately.

    From maps to embeddings

    Real recommendation systems often retrieve neighbors from high-dimensional embeddings learned from content and behavior. This two-axis map is a readable projection of the same core idea: nearby vectors represent similar items.
