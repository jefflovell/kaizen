k-Nearest Neighbors Regression

k-NN predicts a number by finding the most similar known examples and averaging their outcomes.

Business problem

What engagement should we expect before a new title has history?

Situation

A new streaming title has content features but no viewing behavior, creating a cold-start gap.

Problem to solve

The service needs an early numeric estimate before real audience data can replace the proxy.

Why k-NN regression

Similar known titles create a local reference set whose outcomes can be averaged into an intuitive first estimate.

Industry contextTeaching model and local reference method

k-NN makes similarity-based prediction easy to understand and can support retrieval or cold-start systems. It is rarely the default large-scale regressor because prediction requires neighbor search, feature scaling controls the answer, and “near” becomes less meaningful as dimensions grow.

How would a streaming service make sure its new feature film gets shown to the right users when it has no viewing history? A global model could apply one rule to every title, and a tree could place it into a predefined segment. k-NN takes a more local approach: find titles that resemble this one and borrow their signal.

This is useful during cold start, when behavioral data does not exist yet. Content features—tone, pace, genre, runtime, audience, cast, or learned embeddings—provide an initial basis for similarity until real viewing behavior arrives.

    How it is built

    KNN calculates first and predicts second.

    KNN does not learn coefficients or grow a model during training. It stores the known examples, then performs these calculations whenever a new example arrives.

    01 · Measure distance
    d(x, q) = √Σj=1m(xj − qj)2

    Compare the new title q with a known title x across all m features. Σ (“capital sigma”) means “sum,” xj is pronounced “x sub j,” and √ means square root. Here, m = 2.

    02 · Select neighbors
    Nk(q) = k smallest d(xi, q)

    Sort every known title by distance and keep the closest k. Nk(q) is pronounced “N sub k of q” and names the selected neighborhood.

    03 · Predict the target
    ŷ(q) = 1k Σi ∈ Nk(q) yi

    Average the known engagement values yi (“y sub i”) of those neighbors. That average becomes prediction ŷ (“y-hat”) for the new title.

    Key terms

    The vocabulary of local similarity.

    KNN makes predictions from relationships among stored examples, so distance and representation matter enormously.

    Neighbor
    A known example that is close to the query under the chosen distance metric.
    k
    The number of neighbors used for one prediction; pronounced “kay.”
    Euclidean distance
    Straight-line distance calculated from squared feature differences and a square root.
    Feature scaling
    Putting dimensions on comparable scales so one numeric range does not dominate distance.
    Cold start
    Making an initial prediction before clicks, plays, purchases, or other behavioral history exists.
    Lazy learning
    Storing examples during training and postponing most computation until prediction time.
    Embedding
    A learned numeric vector that positions similar items near one another in many-dimensional space.
    Curse of dimensionality
    As dimensions multiply, distance becomes less informative and neighbor search becomes harder.
