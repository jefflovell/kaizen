Signal Bloom

Audio Sampler & Visualizer

Play Signal Bloom View GitHub repo ↗
Sound / Light / Input

Inspiration & purpose

A familiar idea, rebuilt for the modern browser.

Signal Bloom is a modern update of the Patatap clone that I built years ago as part of the Colt Steele Web Developer Bootcamp. Inspired by Patatap, the original idea was simple and joyful: press a key, hear a sound, and see a color.

This version keeps that immediacy while adding a more sophisticated visual character. The envelope shape of each sound drives polygonal explosions, a frequency trace, and a gain meter, turning any keyboard or touchscreen into a small digital sampler.

Tech stack

01

OGL

GPU-rendered fragments, particles, and reactive visuals.

02

Howler

Reliable, cross-browser sample loading and playback.

03

Web Audio

Real-time frequency and energy analysis from each sound.

04

Vite

A fast development loop and a small static production build.

Why this stack

Small tools with clear jobs.

The project needed responsive graphics and dependable audio without the weight of a large application framework. OGL keeps the WebGL layer compact, Howler smooths over browser audio differences, and the native Web Audio API exposes the signal data that connects sound to motion.

Vite ties those pieces together with minimal setup and produces a static site that can be served directly from GitHub Pages. The result stays quick to load, easy to deploy, and focused on the interaction itself.

Source code and sound credits are available in the repository.
